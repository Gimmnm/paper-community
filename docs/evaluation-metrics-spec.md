# Evaluation Metrics Spec

## Output contract

Evaluation overview is generated by `src/analysis_layer/evaluation_metrics.py` and exported to:

- `evaluation_overview.json`
- `evaluation_overview.csv`

## Row-level baseline fields

- `run_id`
- `algorithm`
- `time_window`
- `resolution_min`
- `resolution_max`
- `n_resolution_points`
- `mean_runtime_sec` — mean of per-resolution `time[]` from `summary.npy` (**includes 0** when partitions were loaded from cache).
- `min_runtime_sec`
- `max_runtime_sec`
- `mean_n_communities`
- `max_n_communities`
- `min_n_communities`

### Runtime clarification (Phase A)

- `mean_runtime_active_sec` — mean of `time[k]` where `time[k] > 1e-12` (treat as computed partition); `null` if none computed.
- `n_partitions_cached` — count of resolutions with `time <= 1e-12`.
- `n_partitions_computed` — count with `time > 1e-12`.

Use these when comparing algorithm speed; ignore rows where everything was cache-hit unless you rerun with `--force` / delete memberships.

### Offline plot discovery (Phase A)

Paths are **relative to each manifest’s `leiden_dir`** (POSIX, using `/`):

- `eval_sweep_plot` — e.g. `sweep_diagnostics.png`
- `eval_breakpoints_plot` — first `*breakpoints*.png` or `*breakpoint*.png` in the sweep directory
- `eval_layered_plot` — first `*layered*.png` / `*_layered.svg` / `*cpm_layered*.png`

**Manual override**: `<leiden_dir>/eval/artifacts.json`:

```json
{
  "sweep_diagnostics": "sweep_diagnostics.png",
  "breakpoints": "eval/cpm_breakpoints.png",
  "layered": "figures/domain_0_cpm_layered.svg"
}
```

Keys must match the generator’s expected names; values are relative to `leiden_dir`.

**HTTP**: `GET /api/v3/runs/{run_id}/eval-artifact/{rel_path}` serves a file under that run’s resolved `leiden_dir` (path traversal rejected).

Optional placeholders (filled by downstream evaluators):

- `retrieval_score`
- `topic_score`
- `practical_score`

## Data source

- Primary source: each run's `summary.npy`.
- Registry source: run manifests from `out/experiments/**/manifest.json`.
- Optional scorecard source (if present):
  - `<leiden_dir>/eval/scorecard.json` or sibling `../eval/scorecard.json`
  - keys: `retrieval_score`, `topic_score`, `practical_score`

## API exposure

`GET /api/v3/evaluations/overview` returns merged rows for active catalog.
